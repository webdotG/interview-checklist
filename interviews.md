### Архитектура модуля "Просмотр интервью"

Модуль состоит из нескольких компонентов, каждый из которых отвечает за свою часть функциональности:
- **Управление данными**,
- **Отображение**,
- **Взаимодействие с пользователем**,
- **Вспомогательные сервисы**.

### Описание компонентов

#### 1. `appInitializer.js`
* **Отвечает за:** Начальную загрузку приложения. Он определяет, на какой странице находится пользователь, и запускает соответствующий главный компонент.

#### 2. `InterviewsViewer.js`
* **Отвечает за:** Главный компонент страницы. Он управляет состоянием, загружает данные из базы, обрабатывает события (например, клики по кнопкам), управляет модальными окнами и передает данные другим компонентам для отображения. Это **центр управления**.

#### 3. `UIManager.js`
* **Отвечает за:** Все, что связано с пользовательским интерфейсом. Он управляет состоянием загрузки (`loading`), ошибками (`error-message`), предупреждениями (`local-mode-warning`) и модальным окном. Его задача — отображать пользователю правильное состояние приложения.

#### 4. `InterviewRenderer.js`
* **Отвечает за:** Создание и рендеринг HTML-разметки для карточек интервью и содержимого модального окна. Он берёт данные (объект интервью) и превращает их в готовый HTML, используя вспомогательные сервисы.

#### 5. `InterviewFilters.js`
* **Отвечает за:** Управление логикой фильтрации. Он принимает данные, обрабатывает их на основе выбранных пользователем фильтров и возвращает отфильтрованный список в `InterviewsViewer`.

#### 6. `db.service.js`
* **Отвечает за:** Взаимодействие с базой данных Firebase. Он абстрагирует всю логику подключения и запросов, предоставляя `InterviewsViewer` простые методы для загрузки данных.

#### 7. Вспомогательные сервисы
* `InterviewFormatter.js`: форматирует даты, зарплаты и другие данные для отображения.
* `questions.stats.js`: предоставляет утилиты для подсчета вопросов.
* `questions.data.js`: содержит статические данные о вопросах.

Эта схема наглядно показывает, как каждый компонент выполняет свою роль, работая вместе с другими, чтобы обеспечить единую и согласованную функциональность.
